# LLM-WorldView-Engine 基于大语言模型的世界观引擎

## Motivation 动机

小说，动画，游戏等娱乐产业常常附带很多新奇有趣的世界观设定，但是作为一个小作坊的产品，绝大部分的文学作品世界观都有不够严谨，难以拓展的缺点。就算是创作者自己来续写都很难照顾全部伏笔和设定构思，同人OOC的现象更是司空见惯。而传统依靠读者，编辑，作者的人力筛选很难维护一个完整清晰的世界观，依靠作者个人的知识积累也很难做到彼此照应细节严谨，更没有足够的精力和动力来判断每一段新创作的文本是否符合原来的世界观设定，或者找到原本世界观设定中彼此冲突和不合理的部分。本人最近在玩DeepSeek的时候意外发现这一波大语言模型的想象力非常跳脱，能够在一些规范和引导下构思很多创作者难以完全覆盖的世界观设定。也可以经过阅读理解对比不同同人文本中的世界观设定差异。

以往的大模型虽然可以生成类似的文段，但是想要可靠地提取世界观并对比，就要求大模型本身具备一定基础的理解与推理能力才行。既然这一次DeepSeek R1将推理大模型的价格降低，而且在肉眼可见的将来会有更加便宜和有效的大模型问世，那么我们是否可以在未来几年尝试去构建一个基于大语言模型的世界观引擎呢？

## 目标功能 Target Feature

- 作者全新创作时候的灵感发散和辅助工具
- 作者构建世界观过程中设定冲突的检查工具
- (Optional) 一些形象和设定的可视化工具（借用AI绘图或者传统的DCC管线）
- 其他辅助工具（后端数据库设计，前端简单的操控页面，DCC插件等）
- ....

## 路线图 Roadmap

本质上这还是一个探索性质的长期项目，随时可能因为突发事件发展出现变化，所以暂时也不会做大规模的宣传。

- （3个月内）本人需要自学一段时间大语言模型的基础概念，自己测试一些大语言模型的能力边界，形成一些基础的benchmark
- （半年内）完成一些基础的项目实现，比如收集若干同人文并对比标注其中世界观构思的差异，人物形象的差异等，并手动进行一些标注
- （9个月内）形成一些基础的工具链，进行一些基础的标注 （这一步可能需要一定的志愿者）并对不同的大模型在这一类任务上的能力做一些严谨对比，并尝试打通图文设计的流程
- （1年内）在基于LLM构建的一些新世界观，比如“修仙大明1644”，以及一些成熟世界观（或许有版权问题？）和相关作品中进行一些完善的Demo演示

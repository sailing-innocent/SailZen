## 1. 模块概述

### 1.1 定位与职责

SailZen 服务端与网站是系统的核心数据管理平台，提供：
- **云端数据存储**：基于 PostgreSQL 的持久化存储
- **RESTful API**：为前端和其他客户端提供统一接口
- **Web 用户界面**：响应式设计的管理界面，支持桌面和移动端

### 1.2 技术架构

```
packages/site (前端)
├─ React 18 + TypeScript
├─ Vite (构建工具)
├─ Zustand (状态管理)
├─ TailwindCSS + Radix UI (UI 框架)
├─ Recharts (图表可视化)
└─ React Router (路由)

sail_server (后端)
├─ Python 3.11+
├─ Litestar (Web 框架)
├─ SQLAlchemy (ORM)
├─ PostgreSQL (数据库)
├─ Uvicorn (ASGI 服务器)
└─ Pydantic (数据验证)
```

### 1.3 部署方式

- **后端**：云服务器托管（Uvicorn + Systemd）
- **前端**：静态文件构建后由 sail_server 托管
- **数据库**：云 PostgreSQL 实例
- **访问方式**：通过域名/IP 访问，支持 HTTPS

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 说明 |
|------|-------|------|
| API 响应时间 | P95 < 500ms | 95% 的请求在 500ms 内响应 |
| 页面首屏加载 | < 2s | 包含静态资源加载 |
| 大列表加载 | < 1s | 分页查询响应时间 |
| 并发支持 | 10+ req/s | 单用户场景，峰值并发 |

**优化策略**：
- 数据库索引优化
- 分页查询（前端虚拟滚动）
- API 响应缓存（Redis）
- 静态资源 CDN
- 代码分割与懒加载

### 3.2 可用性需求

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 系统可用性 | 99.5% | 月度可用性 |
| 数据备份 | 每日自动备份 | PostgreSQL 自动备份 |
| RTO (恢复时间) | < 4h | 灾难恢复时间目标 |
| RPO (恢复点) | < 24h | 最大数据丢失时间 |

**保障措施**：
- 云服务器自动备份
- 数据库主从复制
- 监控告警系统
- 灾备预案

### 3.3 安全需求

| 需求 | 实现方式 |
|------|---------|
| 数据传输加密 | HTTPS/TLS |
| 密码存储 | Bcrypt 哈希 |
| SQL 注入防护 | ORM 参数化查询 |
| CSRF 防护 | CSRF Token |
| XSS 防护 | 输入验证 + 输出转义 |
| 访问控制 | （单用户，暂无需求） |

### 3.4 兼容性需求

**浏览器支持**：
- Chrome 90+

## 6. 部署与运维

### 6.1 部署架构

**生产环境**：
```
云服务器 (Ubuntu 22.04)
├─ Nginx (反向代理 + 静态文件服务)
├─ Uvicorn (ASGI 服务器)
│  └─ Litestar App
├─ PostgreSQL (数据库)
└─ Redis (缓存，可选)
```

**部署流程**：
1. 前端构建：`pnpm build-site`
2. 后端部署：复制代码到服务器
3. 安装依赖：`uv sync`
4. 数据库迁移：执行 migration 脚本
5. 启动服务：Systemd 管理 Uvicorn
6. Nginx 配置：反向代理 + 静态文件

### 6.2 监控与告警

**监控指标**：
- 系统资源（CPU、内存、磁盘）
- 应用性能（API 响应时间、错误率）
- 数据库性能（查询时间、连接数）

**告警规则**：
- API 错误率 > 5%
- 响应时间 P95 > 1s
- 磁盘使用率 > 80%
- 数据库连接数 > 80%

**工具**：
- 系统监控：Prometheus + Grafana（可选）
- 日志管理：Logrotate
- 告警通知：邮件/企业微信（可选）

### 6.3 备份与恢复

**备份策略**：
- **数据库备份**：每日自动备份（`pg_dump`）
- **应用代码备份**：Git 仓库
- **配置文件备份**：独立配置仓库
- **备份保留**：保留最近 30 天

**恢复流程**：
1. 停止应用服务
2. 恢复数据库备份（`pg_restore`）
3. 恢复应用代码
4. 检查配置文件
5. 启动服务并验证

## 附录

### A. API 路由总览

**财务模块** (`/api/v1/finance/`)：
- `/account/*` - 账户管理
- `/transaction/*` - 交易管理
- `/budget/*` - 预算管理

**项目模块** (`/api/v1/project/`)：
- `/project/*` - 项目管理
- `/mission/*` - 任务管理

**健康模块** (`/api/v1/health/`)：
- `/weight/*` - 体重管理
- `/fitness/*` - 健身管理（📋 待实现）
- `/care/*` - 保养管理（📋 待实现）
- `/medical/*` - 医疗管理（📋 待实现）

**文本模块** (`/api/v1/text/`)：
- `/work/*` - 作品管理
- `/edition/*` - 版本管理
- `/content/*` - 内容管理

**历史模块** (`/api/v1/history/`)：
- `/event/*` - 历史事件管理

**物资模块** (`/api/v1/necessity/`) - 📋 待实现：
- `/residence/*` - 住所管理
- `/item/*` - 物资管理
- `/inventory/*` - 库存管理
- `/journey/*` - 旅程管理

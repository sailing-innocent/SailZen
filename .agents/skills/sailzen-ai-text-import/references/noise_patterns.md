### URL/链接模式
```regex
https?://[\w\-\.]+\.[a-zA-Z]{2,}[/\w\-\.]*
www\.[\w\-\.]+\.[a-zA-Z]{2,}
[\w\-\.]+\.com
[\w\-\.]+\.cn
[\w\-\.]+\.net
[\w\-\.]+\.org
```

### 典型内容示例


## 格式错误类

### 乱码模式
```
锟斤拷
烫烫烫
屯屯屯
\xFF\xFE
\u0000
```

### 编码问题
- UTF-8 BOM 头残留
- GBK/UTF-8 混编导致的乱码
- 繁简转换错误

### 符号堆积
```regex
^[\s\n\r\t]{10,}$          # 超过10个空白字符
^[\-\=\*]{5,}$              # 超过5个重复符号
^[第\s]{1,}章\s*$           # 只有"第"和"章"的空标题
```

## 重复内容类

### 章节标题重复
- 章节标题在正文中再次出现
- 章节编号重复（如两个"第一章"）

### 固定模板重复
- 每章末尾相同的推荐语
- 每章开头相同的网站名
- 重复的版权声明

## 无意义内容类

### 空内容
- 纯空行
- 仅包含空白字符
- 长度 < 50 字符的"章节"

### 分隔符
```
==========
----------

```

### 占位符
```
（此处省略XX字）

```

## 清理规则汇总

### 自动清理（高置信度）
1. URL 链接
2. 邮箱地址
3. 纯符号行（>10个重复符号）
4. 纯空白行（>5个连续空行）
5. 明确广告关键词行

### 标记警告（需人工确认）
1. 长度 < 200 字符的"章节"
2. 包含"关注"、"扫码"等词汇但可能是正文内容
3. 疑似重复的章节标题
4. 异常的章节顺序（如第10章后紧跟第5章）

### 保留但标记
1. 作者的话（标记为 author 类型）
2. 请假条（标记为 notice 类型）
3. 更新时间说明（标记为 meta 类型）

## 正则表达式库

### Python 清理用正则

```python
# URL 匹配
URL_PATTERN = re.compile(
    r'https?://[^\s<>"{}|\\^`\[\]]+'
    r'|www\.[^\s<>"{}|\\^`\[\]]+'
    r'|[a-zA-Z0-9.-]+\.(com|cn|net|org|io|cc)',
    re.IGNORECASE
)

# 广告关键词行
AD_KEYWORDS = [
    r'关注.*公众号',
    r'扫码.*加群',
    r'加入.*QQ群',
    r'关注.*微博',
    r'关注.*抖音',
    r'正版.*订阅',
    r'感谢.*投票',
    r'求推荐',
    r'求收藏',
    r'求月票',
]
AD_PATTERN = re.compile('|'.join(AD_KEYWORDS))

# 纯符号行
SYMBOL_PATTERN = re.compile(r'^[\s\-=\*#~]{10,}$')

# 乱码检测（常见乱码字符）
GARBAGE_CHARS = re.compile(r'[锟斤拷烫屯臺]')

# 空内容
EMPTY_PATTERN = re.compile(r'^[\s\n\r\t]{0,100}$')
```

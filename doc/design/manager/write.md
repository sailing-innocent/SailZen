## 1. 快速开始

### 1.1 前置条件

- 确保后端服务已启动
- 确保数据库表已创建（首次启动会自动创建）

### 1.2 访问文本管理页面

打开浏览器访问：`http://localhost:1974/text`

### 1.3 导入第一本小说

1. 点击页面右上角的 **"导入文本"** 按钮
2. 填写作品标题和作者信息
3. 上传 `.txt` 文件或直接粘贴文本内容
4. 点击 **"开始导入"**
5. 系统会自动识别章节结构并导入

## 2. 功能概述

### 2.1 核心概念

- **作品 (Work)**：一本书或一部小说
- **版本 (Edition)**：作品的一个具体版本（如原版、译版、修订版）
- **文档节点 (DocumentNode)**：树形结构存储的文本内容单元（如章节、段落）

### 2.2 支持的章节格式

系统自动识别以下章节格式：

| 格式 | 示例 |
|------|------|
| 中文章节 | `第一章 风起云涌`、`第1章 开始` |
| 中文节 | `第一节`、`第二节` |
| 英文章节 | `Chapter 1 The Beginning` |
| 数字章节 | `1. Introduction` |
| 方括号标题 | `【序章】`、`【第一章】` |

## 3. 导入小说

### 3.1 通过界面导入

1. **打开导入对话框**：点击 "导入文本" 按钮
2. **填写基本信息**：
   - 作品标题（必填）
   - 作者名
   - 语言（中文/英文）
   - 作品简介
3. **上传或粘贴内容**：
   - 方式一：点击上传区域选择 `.txt` 或 `.md` 文件
   - 方式二：直接在文本框中粘贴内容
4. **确认导入**：点击 "开始导入"

### 3.2 渐进式导入（追加章节）

如果小说还在连载，可以追加新章节：

```bash
# API 方式追加章节
POST /api/v1/text/import/append/{edition_id}
Content-Type: application/x-www-form-urlencoded

content=第一百零一章 新的开始...
```

### 3.3 导入大型小说的建议

对于超大文本（>100MB），建议：

1. **分批导入**：先导入前100章，确认格式正确后再追加
2. **自定义章节模式**：如果默认模式无法识别，可以提供正则表达式
3. **预处理文本**：确保章节标题格式统一

## 4. 查看内容

### 4.1 作品列表

- 主页面显示所有已导入的作品
- 显示信息：标题、作者、章节数、字数、状态
- 支持搜索：在搜索框输入关键词搜索作品

### 4.2 阅读章节

1. 点击作品卡片进入阅读界面
2. 左侧显示章节目录（可点击跳转）
3. 右侧显示章节内容
4. 使用底部按钮切换上一章/下一章

### 4.3 搜索内容

在版本内搜索特定内容：

```bash
GET /api/v1/text/edition/{edition_id}/search?keyword=关键词
```

## 5. 编辑章节

### 5.1 在界面中编辑

1. 在阅读界面，点击右上角 **"编辑"** 按钮
2. 修改章节标题或内容
3. 点击 **"保存"** 确认更改

### 5.2 通过 API 编辑

```bash
PUT /api/v1/text/node/{node_id}
Content-Type: application/json

{
  "title": "新标题",
  "raw_text": "修改后的内容..."
}
```

### 5.3 编辑注意事项

- 修改会自动更新字数统计
- 支持撤销（点击"取消"按钮）
- 建议在编辑前备份重要内容

## 6. API 参考

### 6.1 作品管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/text/work` | 获取作品列表 |
| GET | `/api/v1/text/work/{id}` | 获取单个作品 |
| POST | `/api/v1/text/work/` | 创建作品 |
| PUT | `/api/v1/text/work/{id}` | 更新作品 |
| DELETE | `/api/v1/text/work/{id}` | 删除作品 |
| GET | `/api/v1/text/work/search?keyword=xxx` | 搜索作品 |

### 6.2 版本管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/text/edition/{id}` | 获取版本详情 |
| GET | `/api/v1/text/edition/work/{work_id}` | 获取作品的所有版本 |
| GET | `/api/v1/text/edition/{id}/chapters` | 获取章节目录 |
| GET | `/api/v1/text/edition/{id}/chapter/{index}` | 获取章节内容 |
| GET | `/api/v1/text/edition/{id}/search?keyword=xxx` | 搜索版本内容 |

### 6.3 导入

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/text/import/` | 导入新文本 |
| POST | `/api/v1/text/import/append/{edition_id}` | 追加章节 |

### 6.4 文档节点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/text/node/{id}` | 获取节点详情 |
| PUT | `/api/v1/text/node/{id}` | 更新节点 |

## 7. 常见问题

### Q1: 章节没有被正确识别？

**A**: 检查章节标题格式是否符合支持的模式。如果使用特殊格式，可以在导入时提供自定义正则表达式：

```json
{
  "chapter_pattern": "^===\\s*(.+)\\s*===$"
}
```

### Q2: 导入后显示只有1章？

**A**: 可能是因为：
1. 文本中没有章节标记
2. 章节标记格式不被识别

解决方案：
- 手动添加章节标记后重新导入
- 使用自定义章节模式

### Q3: 如何删除导入错误的作品？

**A**: 在作品列表中，点击作品卡片右下角的 "删除" 按钮。**注意：此操作不可恢复。**

### Q4: 支持什么文件格式？

**A**: 目前支持：
- `.txt` - 纯文本文件
- `.md` - Markdown 文件

建议使用 UTF-8 编码。

### Q5: 最大支持多大的文件？

**A**: 理论上没有硬性限制，但建议：
- 单次导入 < 50MB
- 超大文件分批导入

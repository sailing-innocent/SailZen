# SailZen Multi-Agent 系统架构设计

## 系统愿景

构建一个**分布式、可扩展、高可用**的多Agent系统，通过飞书Bot作为统一入口，接管机器上多个子项目的开发、Review、构建、维护、上线全流程。

## 核心目标

| 目标 | 描述 | 关键指标 |
|------|------|----------|
| **统一入口** | 通过飞书Bot自然语言交互 | 支持100+种意图识别 |
| **多项目管理** | 同时管理多个代码仓库 | 支持并发10+项目 |
| **持久化** | 数据库持久化所有状态 | 100%状态可追溯 |
| **断点续传** | 支持任意点恢复执行 | 恢复时间<5s |
| **工作区隔离** | 项目间完全隔离 | 零干扰 |
| **可扩展** | 插件化Agent架构 | 新Agent接入<1天 |

```yaml
workflow:
  name: "feature-development"
  description: "新功能开发完整流程"
  
  steps:
    - id: "analyze"
      name: "需求分析"
      agent: "analyze_agent"
      input:
        requirement: "{{user_input}}"
      output:
        analysis_report: "report"
    
    - id: "design"
      name: "方案设计"
      agent: "design_agent"
      depends_on: ["analyze"]
      input:
        report: "{{steps.analyze.output.report}}"
      output:
        design_doc: "doc"
    
    - id: "implement"
      name: "代码实现"
      agent: "code_agent"
      depends_on: ["design"]
      input:
        design: "{{steps.design.output.doc}}"
      output:
        code_changes: "changes"
    
    - id: "review"
      name: "代码审查"
      agent: "review_agent"
      depends_on: ["implement"]
      input:
        changes: "{{steps.implement.output.changes}}"
      output:
        review_report: "report"
      
      # 条件分支
      on_complete:
        - condition: "{{output.review_report.score}} > 80"
          next: "test"
        - condition: "{{output.review_report.score}} < 60"
          next: "implement"  # 打回重改
        - default: "fix"
    
    - id: "fix"
      name: "代码修复"
      agent: "code_agent"
      input:
        changes: "{{steps.implement.output.changes}}"
        review: "{{steps.review.output.report}}"
      output:
        fixed_changes: "changes"
      next: "review"  # 重新审查
    
    - id: "test"
      name: "测试执行"
      agent: "test_agent"
      depends_on: ["review"]
      parallel: true  # 并行执行多个测试
      steps:
        - id: "unit_test"
          agent: "test_agent"
          config:
            test_type: "unit"
        - id: "integration_test"
          agent: "test_agent"
          config:
            test_type: "integration"
      
      aggregator: "merge_test_results"
    
    - id: "deploy"
      name: "部署上线"
      agent: "deploy_agent"
      depends_on: ["test"]
      confirmation_required: true  # 需要人工确认
```

### 6. Checkpoint & Recovery (检查点与恢复)

**检查点策略：**

```python
class CheckpointManager:
    """检查点管理器 - 支持断点续传"""
    
    async def create_checkpoint(
        self,
        workflow_id: str,
        step_id: str,
        state: WorkflowState
    ) -> Checkpoint:
        """创建检查点"""
        checkpoint = Checkpoint(
            id=generate_uuid(),
            workflow_id=workflow_id,
            step_id=step_id,
            state=state.serialize(),
            timestamp=datetime.now(),
            metadata={
                "memory_usage": get_memory_usage(),
                "disk_usage": get_disk_usage(),
            }
        )
        await self.persistence.save(checkpoint)
        return checkpoint
    
    async def restore_from_checkpoint(self, checkpoint_id: str) -> WorkflowState:
        """从检查点恢复"""
        checkpoint = await self.persistence.load(checkpoint_id)
        return WorkflowState.deserialize(checkpoint.state)
```

**自动保存策略：**

| 触发条件 | 保存内容 | 保留策略 |
|----------|----------|----------|
| 步骤完成 | 完整状态 | 保留最近10个 |
| 异常中断 | 完整状态 + 堆栈 | 永久保留 |
| 定时保存 | 增量状态 | 保留最近24小时 |
| 手动保存 | 完整状态 | 永久保留 |

## 技术栈选型

| 组件 | 选型 | 理由 |
|------|------|------|

| **容器化** | Docker | 工作区隔离 |
| **监控** | Prometheus + Grafana | 指标收集与可视化 |
| **日志** | ELK Stack / Loki | 集中式日志管理 |

## 总结

本架构设计遵循以下核心原则：

1. **分层清晰**: 每层职责明确，便于独立扩展和维护
2. **高可用**: 关键组件多实例部署，支持故障转移
3. **可观测**: 全链路监控、日志、追踪
4. **安全性**: 认证授权、审计日志、敏感信息保护
5. **可扩展**: 插件化Agent架构，新功能快速接入

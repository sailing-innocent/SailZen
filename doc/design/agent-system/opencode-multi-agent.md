# SailZen Multi-Agent System 设计文档

## 概述

本文档是SailZen Multi-Agent系统的完整设计文档，旨在构建一个**分布式、可扩展、高可用**的多Agent系统，通过飞书Bot作为统一入口，接管机器上多个子项目的开发、Review、构建、维护、上线全流程。

## 系统愿景

构建一个真正的**AI驱动的开发运维助手**，实现：

- **统一入口**: 通过飞书Bot自然语言交互，支持100+种意图识别
- **多项目管理**: 同时管理多个代码仓库，支持并发10+项目
- **持久化**: 数据库持久化所有状态，100%状态可追溯
- **断点续传**: 支持任意点恢复执行，恢复时间<5s
- **工作区隔离**: 项目间完全隔离，零干扰
- **可扩展**: 插件化Agent架构，新Agent接入<1天

### 核心设计理念

#### 1. 事件驱动架构
```
所有组件通过事件总线通信，松耦合、可扩展
Event Bus (Redis Streams) → 发布/订阅模式
```

#### 2. 状态外部化
```
Agent无状态，所有状态持久化到数据库
支持任意点恢复（断点续传）
```

#### 3. 分层隔离
```
工作区级别隔离：进程、端口、文件系统
支持Docker容器级隔离
```

#### 4. 插件化Agent
```
新Agent通过注册中心动态接入
能力自动发现与匹配
```

## 技术栈

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| **API框架** | Litestar | 高性能ASGI，已使用 |
| **数据库** | PostgreSQL | 复杂查询支持，已使用 |
| **缓存** | Redis | 会话、消息队列、分布式锁 |
| **消息队列** | Redis Streams | 统一技术栈 |
| **序列化** | msgpack + zlib | 高效、紧凑 |
| **工作流** | 自研 | 灵活、可控 |
| **Agent通信** | gRPC / HTTP2 | 高性能 |
| **容器化** | Docker | 隔离 |
| **监控** | Prometheus + Grafana | 成熟方案 |

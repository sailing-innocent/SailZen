## 常见边界场景

### 需要跳过的交易

以下类型的交易通常不打标签或需要人工判断：

| 场景 | description 示例 | 建议 |
|------|-----------------|------|
| 账户间转账 | "余额宝转出"、"信用卡还款" | 跳过（from 和 to 都 > 0） |
| 含糊不清 | "转账"、"消费"、"支出" | 跳过，留给人工 |
| 纯数字/编号 | "20240601"、"#12345" | 跳过 |
| 收入交易 | "工资"、"报销"、"退款" | 跳过或用 income 标签（如已创建） |

### 可能歧义的交易

| description | 可能标签 | 推荐策略 |
|-------------|---------|----------|
| "超市" | 零食 or 日用消耗 | 看金额：< 50 大概率零食，> 100 大概率日用 |
| "淘宝" | 日用消耗 or 衣物 or 大宗电器 | 无法确定，跳过 |
| "微信转账" | 人际交往 or 其他 | 无法确定，跳过 |
| "会员充值" | 娱乐休闲 or 日用消耗 | 看具体内容描述 |

## 批量写入的最佳实践

### 分批处理

```
第1轮: 处理所有能精确匹配的（confidence: high）
第2轮: 处理关键词匹配的（confidence: medium）
第3轮: 展示剩余无法处理的，让用户决定
```

### 报告格式

每批处理后，清晰报告：
1. ✅ 成功打标的数量和分布
2. ⏭️ 跳过的交易及原因
3. ❌ 失败的交易及错误信息
4. 📊 累计进度（已处理/总数）

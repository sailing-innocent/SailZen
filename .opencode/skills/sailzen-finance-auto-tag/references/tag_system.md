# 标签体系参考

## 标签分类

SailZen 财务系统的标签分为以下几个分类：

### expense（日常支出）

日常开销类标签，使用频率最高。

| 标签名 | 颜色 | 适用关键词 | 典型 description |
|--------|------|-----------|-----------------|
| 零食 | #8884d8 | 外卖、饮料、奶茶、小吃、食堂、餐厅 | "美团外卖"、"瑞幸咖啡"、"食堂午餐" |
| 交通 | #82ca9d | 打车、滴滴、地铁、公交、高铁、机票 | "滴滴出行"、"地铁充值"、"12306" |
| 日用消耗 | #ffc658 | 超市、日用品、洗护、文具、网购日用 | "京东购物"、"超市采购"、"淘宝" |
| 娱乐休闲 | #00ff00 | 电影、游戏、运动、旅游、景点门票 | "电影票"、"Steam充值"、"游泳卡" |
| 人际交往 | #ff00ff | 聚餐、礼物、红包、份子钱 | "同事聚餐"、"生日礼物"、"微信红包" |
| 医药健康 | #00ffff | 药品、就医、体检、保健品 | "药店买药"、"挂号费"、"体检" |
| 衣物 | #ff0080 | 衣服、鞋子、帽子、配饰 | "优衣库"、"Nike"、"运动鞋" |

### major（大宗支出）

大额或低频支出。

| 标签名 | 颜色 | 适用关键词 | 典型 description |
|--------|------|-----------|-----------------|
| 大宗电器 | #ff7300 | 家电、手机、电脑、平板、相机 | "Apple Store"、"小米商城"、"洗衣机" |
| 大宗收支 | #8000ff | 房租、学费、保险、大额转账 | "房租"、"学费"、"保险年缴" |

### income（收入）

收入类标签（可能需要用户后续创建）。

| 建议标签 | 适用场景 |
|----------|----------|
| 工资 | 月薪、奖金 |
| 报销 | 公司报销 |
| 投资收益 | 理财、基金收益 |
| 其他收入 | 退款、二手转卖 |

> 注意：收入类标签默认未创建，Agent 遇到收入交易时应跳过或建议用户创建。

## 标签与交易方向的关系

| 交易方向 | from_acc_id | to_acc_id | 含义 | 常用标签分类 |
|----------|-------------|-----------|------|-------------|
| 支出 | > 0 | -1 | 钱流出到外部 | expense, major |
| 收入 | -1 | > 0 | 钱从外部流入 | income |
| 转账 | > 0 | > 0 | 账户间转移 | 通常不打标签 |

## 标签数据存储

标签以逗号分隔的字符串存储在 `transactions.tags` 字段中。

**示例：**
```
"零食"              → 单标签
"交通,日用消耗"     → 多标签（极少见）
""                   → 无标签（待打标）
null                 → 无标签（历史数据）
```

## 已注册 vs 野生标签

- **已注册标签**：在 `finance_tags` 表中有记录，有颜色/分类/排序等元数据
- **野生标签**：在交易的 `tags` 字段中使用过，但未在 `finance_tags` 表中注册
- `GET /finance/tag/stats` 接口会区分报告两者
- 建议：发现野生标签后，通过 `POST /finance/tag/` 将其注册为正式标签
